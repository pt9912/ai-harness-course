# Uebung 08 — Implementation-Agent gefuehrt laufen lassen

## Ziel

Du laesst einen Implementation-Agent nicht frei codieren, sondern durch
den 8-Schritt-Workflow laufen.

## Startzustand

Nutze den abgeschlossenen Slice als Referenz:

- [`../docs/plan/planning/done/slice-009-tie-break-determinismus.md`](../docs/plan/planning/done/slice-009-tie-break-determinismus.md)
- [`../AGENTS.md`](../AGENTS.md)
- [`../harness/README.md`](../harness/README.md)

## Aufgabe

1. Fuehre `make agent-implement SLICE=slice-009` im Verzeichnis
   `lab/example/` aus.
2. Notiere, welche Quellen der Agent mindestens lesen muss.
3. Formuliere einen Plan vor Code fuer eine analoge Aenderung in deinem
   eigenen Uebungs-Repo.
4. Streiche absichtlich `AGENTS.md` aus dem Kontext und beschreibe,
   welche Fehlannahme dadurch wahrscheinlicher wird.

## Erwartetes Zwischenprodukt

Ein sichtbarer Plan mit betroffenen IDs, engstem Sensor und
repo-weitem Abschlussgate. Ohne Plan ist die Uebung nicht bestanden.
